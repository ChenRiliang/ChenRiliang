小程序踩坑记

发表于 2017-10-25 | 分类于 小程序 | | 阅读次数:

初次接触小程序，并在这周完成了公司项目，在这里统一总结下关于小程序和IDE中遇到的坑:


1. 怎么对data进行赋值

设置数据时不能直接进行打点赋值，而应利用 this.setData 如:

this.setData({
    name: 'Joker'
})

2. 小程序取值 this.name 不成功

小程序语法以及整体框架类似于MVVM如Vue但是又存在着差异:

this.data.name

3. 设置跳转至tabBar页面报错

底栏导航栏可直接在app.json上进行设置 tabBar，并且tabBar上的页面跳转不能使用 wx.navigateTo(OBJECT) 而应该使用 wx.switchTab(OBJECT)

4. 小程序能使用 less/sass 吗 ？

官方 IDE 暂时还没有提供 less 和 sass 等 css 预编译器支持，不建议使用。 官方 102800 已开始支持 css 的 @import 语法，推荐使用， 推荐参考 weui-wxss 进行 wxss 模块化。

5. image 高度设为 auto？

原先height:auto失效,必须指定image的高度为具体数值,不然高度为0.

6. canvas/view z-index 失效 ?

canvas 组件是由客户端创建的原生组件，它的层级是最高的,即会导致其他标签z-index失效无法在其之上。

7. canvas 设置 tranform 缩放、旋转失效 ？

canvas旋转缩放可用wxcanvas封装好的API 如:canvasContext.rotate,另外 css 动画对 canvas 组件无效。

也可能是小case ， 后续再补充下。
